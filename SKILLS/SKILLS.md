# Skills Directory

This directory contains skills available for use by OpenCode agents when building this project.

## Available Skills

### [frontend-design](./frontend-design.md)
**Purpose**: Create distinctive, production-grade frontend interfaces with high design quality

**When to use**:
- Building web components, pages, or applications
- Creating UI designs for sinking fund platform
- Designing user interfaces for any web application
- Implementing visual improvements or redesigns

**Key Features**:
- Guidance on choosing bold aesthetic directions
- Typography, color, and motion guidelines
- Avoiding generic AI-generated aesthetics
- Component examples with code
- Implementation checklist

**How to use**:
1. Read the full skill documentation
2. Understand project context and requirements
3. Choose an aesthetic direction (minimalist, maximalist, etc.)
4. Execute with precision and creativity
5. Follow anti-patterns to avoid generic design

---

## Using Skills

### For Agents

1. **Read the skill** - Full documentation before starting
2. **Understand context** - Project requirements, audience, constraints
3. **Apply skill** - Follow guidelines and examples
4. **Be creative** - Don't hold back, make it memorable
5. **Test thoroughly** - Ensure production-grade quality

### For Users

When requesting work:
- Mention the skill name if you have a preference
- Example: "Use the frontend-design skill to build the dashboard"
- Or agents will choose appropriate skill automatically

---

## Adding New Skills

To add a new skill:

1. Create a new markdown file in `/SKILLS/` directory
2. Follow the skill template:
   ```markdown
   ---
   name: skill-name
   description: Brief description
   license: Complete terms in LICENSE.txt
   ---
   
   ## Overview
   [Documentation here]
   ```
3. Update this `SKILLS.md` to list the new skill
4. Add examples and guidelines
5. Include implementation checklist

---

## Best Practices

- **Keep skills focused** - Each skill should have clear purpose
- **Provide examples** - Code snippets help agents implement correctly
- **Include checklists** - Verify completeness
- **Avoid conflicts** - Skills should complement, not overlap
- **Version control** - Update when improvements are made
